    • SQL Optimization

    This week we were introduced to databases and structured query language or SQL for short. SQL statements are used to retrieve data from database. In order to retrieve data you need to access the data by writing SQL queries. SQL queries can be written by using SQL keywords and by following a set of rules. In this blog, I am going to discuss best practices and compare using some example to show how one can improve the efficacy of a query.

    Example 1: All Vs selecting actual columns

    By selecting the actual columns the time required to return an answer decreases, otherwise the query will have to pull all the columns even though you only require information from a few. The general rule is to avoid full table scans as much as possible, especially for big tables.

    Example 2: Try to avoid the use of !not operator

    It is much faster to return a correct match by using = or IN rather than not != query. When you use a negative operator, it will have to check every row in the table, which can be time consuming. With the use of = operator, it will make the query find that particular row, which meets the search requirement.

    Example 3: Use the HAVING clause just as filter

    Save using HAVING clause to "filter", this will make your query more efficient.

    Example 4: Wildcard

    Using a wildcard on large tables will slow your query, but if you have to use it try adding it using prefix or postfix. This is make the query comparatively faster.

    Example 5: AND Vs OR

    Using OR will cause the query to check all the rows, while the use of AND will be more precise requiring less time
